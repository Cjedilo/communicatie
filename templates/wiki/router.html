Router setup & port forwarding

To let people outside your home network reach your server, you need to tell your router to forward incoming traffic to the machine running Messages.

What it is Your local IP Static local IP Port forwarding Test it Common routers

What is port forwarding?

Your internet connection has one public IP address shared by all devices in your home. When an outside visitor connects to your server, the connection arrives at your router. Without port forwarding, the router doesn't know which device to pass it to, so it drops it.

Port forwarding creates a rule: traffic arriving on port X → send it to this machine on this internal IP.

Step 1 — Find your machine's local IP

The local IP is the address your router assigned to the machine running Messages. It usually looks like 192.168.1.x or 10.0.0.x.

OS Command
Linux ip addr show | grep "inet "
Windows ipconfig — look for IPv4 Address
macOS ifconfig | grep "inet "

Write it down — you'll need it in the port forwarding rule.

Step 2 — Give the machine a static local IP

By default, routers hand out IP addresses dynamically — your machine might get a different address after a reboot. If that happens, your port forwarding rule points at the wrong device.

The easiest fix is a DHCP reservation (also called static DHCP or IP binding): you tell the router to always give this machine the same address, based on its MAC address. No changes needed on the machine itself.

In your router admin panel, look for:

  • DHCP reservations
  • Static IP assignment
  • IP & MAC binding
  • Address reservation

Add a reservation for your machine's MAC address and assign it the IP you found in step 1.

Step 3 — Create a port forwarding rule

Log into your router admin panel. The address is usually 192.168.1.1 or 192.168.0.1 — type it in a browser. If that doesn't work, run ip route | grep default (Linux) or look at the Default Gateway in ipconfig (Windows).

Navigate to the port forwarding section. Common names:

  • Port Forwarding
  • Virtual Server
  • NAT / PAT
  • Applications & Gaming

Create a new rule with these values:

FieldValue
Service / NameMessages
ProtocolTCP
External port443 (or the port you chose during install)
Internal IPThe local IP of your machine
Internal portSame as external port

Save and apply. Some routers need a reboot to activate rules.

If you're also using port 80 for the HTTP redirect, add a second rule with the same internal IP, protocol TCP, and port 80.

Step 4 — Test it

Find your public IP at a site like ifconfig.me (just open it in a browser). Then try opening https://<your-public-ip>:443 from a device on a different network — a phone with Wi-Fi turned off works well.

If it works: great. Set up a domain name to avoid sharing a raw IP address with people.

If it doesn't:

  • Check that Messages is actually running (systemctl status messages on Linux)
  • Check that the firewall on the machine allows the port (sudo ufw allow 443/tcp on Ubuntu)
  • Some ISPs block port 443 on residential connections — try port 8443 instead

Common routers

ASUS

Go to WAN → Virtual Server / Port Forwarding. Click Add profile. Fill in service name, protocol (TCP), external and internal port, and the local IP. Click OK, then Apply.

For DHCP reservation: LAN → DHCP Server → Manually Assigned IP.

TP-Link (Archer)

Go to Advanced → NAT Forwarding → Virtual Servers. Click Add. Fill in external and internal port, protocol TCP, and internal IP. Save.

For DHCP reservation: Advanced → Network → DHCP Server → Address Reservation.

FRITZ!Box

Go to Internet → Permit Access → Port Sharing. Click New port sharing. Choose Other application, fill in a name, select the device, set protocol to TCP and both ports to your chosen value. Click OK.

FRITZ!Box assigns static local IPs automatically when you set up port sharing — select Always assign this network device the same IPv4 address when prompted.

Netgear

Go to Advanced → Advanced Setup → Port Forwarding / Port Triggering. Click Add Custom Service. Fill in name, TCP, start and end port (same number), and the internal IP. Click Apply.

For DHCP reservation: Advanced → Setup → LAN Setup → Address Reservation → Add.

Ziggo Connect Box / UPC

Go to Advanced → Security → Port Forwarding. Add a rule with the local IP, TCP, and the port number for both start and end. Save.

KPN Experia Box

Go to Internet → Poort Doorsturen. Add a new rule for TCP, the chosen port, and the local IP of your machine. Save.

Still stuck? The site portforward.com has step-by-step guides for hundreds of specific router models.
